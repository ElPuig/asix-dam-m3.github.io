L'objectiu d'aquest CodeLab és conèixer les possibilitats de la classe Scanner per a llegir dades d'entrada des d'un fitxer.

En molts casos, quan provem un programa, ens veiem obligat a introduir per teclat una vegada i una altra les mateixes dades. Això ralenteix el nostre treball. En aquest CodeLab veurem com posar aquestes dades en un fitxer i llegir-les directament.

Inicia IntelliJ i crea un nou projecte anomenat "ScannerFile".

Imagina que estàs tractant de fer un programa per a gestionar alumnes. De moment, tens una mica de codi per mostrar un menu d'opcions, per a donar d'alta alumnes, i llistar els alumnes.

import java.util.Scanner;

class Alumne {
   String nom, cognoms;
}

public class Main {

   public static void main(String[] args) {
       Scanner scanner = new Scanner(System.in);

       Alumne[] alumnes = new Alumne[10];
       int posicio = 0;

       while(true) {
           System.out.println();
           System.out.println("GESTIO ALUMNES");
           System.out.println("a) Introduir alumne");
           System.out.println("b) Llistar alumnes");
           System.out.println();

           String opcio = scanner.nextLine();

           switch (opcio) {
               case "a":
                   alumnes[posicio] = new Alumne();

                   System.out.println("Nom: ");
                   alumnes[posicio].nom = scanner.nextLine();
                   System.out.println("Cognoms: ");
                   alumnes[posicio].cognoms = scanner.nextLine();

                   posicio++;
                   break;
               case "b":
                   System.out.println("Llistat d'alumnes");
                   for (int i = 0; i < posicio; i++) {
                       System.out.println(alumnes[i].cognoms + ", " + alumnes[i].nom);
                   }
                   break;
               default:
                   return;
           }
       }
   }
}

Cada cop que fas una modificació el codi, executes el programa i introdueixes les dades per veure com va funcionant el programa.

Això significa una valuosa pèrdua de temps, que podem evitar posant les dades d'entrada en un arxiu i llegint les dades directament de l'arxiu
