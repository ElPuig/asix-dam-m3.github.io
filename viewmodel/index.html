El objetivo de esta práctica es explicar el elemento ViewModel.

La clase ViewModel está diseñada para almacenar y gestionar datos relacionados con la UI de forma consciente al ciclo de vida. La clase ViewModel permite que los datos sobrevivan a cambios de configuración, como la rotación de la pantalla.

En esta práctica ampliaremos la practica anterior: https://gerardfp.github.io/recyclerview/ de forma que permita valorar con estrellas cada poema.

Realiza una copia del proyecto https://gerardfp.github.io/recyclerview/

Añade el componente Lifecycle en las dependencias build.gradle (Module: app):

implementation "android.arch.lifecycle:extensions:1.1.1"
annotationProcessor "android.arch.lifecycle:compiler:1.1.1"

Obtén los números de versión actualizados para ViewModel: adding components to your project.

Añade a la clase Poem el campo rating:

public class Poem {
   public String title;
   public String author;
   String content;
   float rating;
}

Añade un elemento <RatingBar> en el layout item_poem.xml:

<RatingBar
   android:id="@+id/ratingBar"
   android:layout_width="wrap_content"
   android:layout_height="wrap_content" />

Modifica el adaptador para incluir el elemento RatingBar:

public class PoemsRecyclerAdapter extends RecyclerView.Adapter<PoemsRecyclerAdapter.PoemViewHolder>{

   List<Poem> list;

   PoemsRecyclerAdapter(List<Poem> list){
       this.list = list;
   }

   @Override
   public PoemViewHolder onCreateViewHolder(ViewGroup parent, int viewType) {
       View itemPoem = LayoutInflater.from(parent.getContext()).inflate(R.layout.item_poem, parent, false);
       return new PoemViewHolder(itemPoem);
   }

   @Override
   public void onBindViewHolder(PoemViewHolder holder, int position) {
       final Poem poem = list.get(position);

       holder.poemAuthor.setText(poem.author);
       holder.poemTitle.setText(poem.title);
       holder.ratingBar.setRating(poem.rating);
       holder.ratingBar.setOnRatingBarChangeListener(new RatingBar.OnRatingBarChangeListener() {
           @Override
           public void onRatingChanged(RatingBar ratingBar, float v, boolean b) {
               if(b) poem.rating = v;
           }
       });
   }

   @Override
   public int getItemCount() {
       return (list != null ? list.size() : 0);
   }

   class PoemViewHolder extends RecyclerView.ViewHolder {
       private TextView poemTitle;
       private TextView poemAuthor;
       private RatingBar ratingBar;

       PoemViewHolder(View itemPoem) {
           super(itemPoem);
           poemTitle = itemPoem.findViewById(R.id.poem_title);
           poemAuthor = itemPoem.findViewById(R.id.poem_author);
           ratingBar = itemPoem.findViewById(R.id.ratingBar);
       }
   }
}
  1. En el ViewHolder hemos hecho la búsqueda del elemento R.id.ratingBar, guardándola en la variable ratingBar.
  2. En el método onBindViewHolder(), rellenamos el RatingBar con las estrellas que tenga cada poema, y añadimos un listener para actualizar el rating de cada poema.

Al ejecutar la App observa que cuando se cambia el rating de un poema, podemos hacer scroll en la lista y el rating se mantiene. Sin embargo, si rotamos el móvil los ratings se pierden. Esto es debido a que al hacer la rotación la MainActivity se destruye y se vuelve a crear, por tanto el ArrayList también.

Para prevenir que el ArrayList se pierda al hacer la rotación del móvil, usaremos un ViewModel, ya que este sobrevive a la rotación.

Crea la clase PoemsViewModel, de forma que contenga el array de poemas y un método para obtenerlo: 

public class PoemsViewModel extends ViewModel {
   List<Poem> poemList = new ArrayList<>();

   List<Poem>  getPoemList(){
       return poemList;
   }
}

Modificamos la MainActivity para que obtenga el array de poemas del ViewModel:

public class MainActivity extends AppCompatActivity {

   List<Poem> poemList;

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       PoemsViewModel poemsViewModel = ViewModelProviders.of(this).get(PoemsViewModel.class);

       poemList = poemsViewModel.getPoemList();

       for(int i = 0; i<200; i++){
           Poem poem = new Poem();
           poem.author = "Author " + i;
           poem.title = "Title " + i;
           poemList.add(poem);
       }

       RecyclerView recyclerView = findViewById(R.id.poem_list);
       recyclerView.setLayoutManager(new LinearLayoutManager(this));

       PoemsRecyclerAdapter poemsRecyclerAdapter = new PoemsRecyclerAdapter(poemList);
       recyclerView.setAdapter(poemsRecyclerAdapter);
   }
}

NOTA: Si en el ViewModel se necesita el contexto de la Application (por ejemplo para obtener una base de datos Room), se puede extender de la clase AndroidViewModel obteniendo así en el constructor una referencia la Application.

public class PoemsViewModel extends AndroidViewModel {

Context context;

public PoemsViewModel(@NonNull Application application) {

super(application);

context = application;

}

}

Ampliaremos la aplicación para que al hacer click en un poema de la lista, se abra otra Activity y lo muestre.

Empezaremos creando la Activity que muestra un poema. Esta PoemActivity recibirá en el Intent la posición en el array del Poema que se debe mostrar. A continuación extraerá el ArrayList de poemas del ViewModel y mostrará los datos de dicho poema.

public class PoemActivity extends AppCompatActivity {

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_poem);

       List<Poem> poems = ViewModelProviders.of(this).get(PoemsViewModel.class).getPoemList();

       int poemId = getIntent().getIntExtra("POEM_ID",0);

       Poem poem = poems.get(poemId);

       if(poem != null) {
           ((TextView) findViewById(R.id.poem_title)).setText(poem.title);
           ((TextView) findViewById(R.id.poem_author)).setText(poem.author);
           ((RatingBar) findViewById(R.id.poem_rating)).setRating(poem.rating);
           ((TextView) findViewById(R.id.poem_content)).setText(poem.content);
       }
   }
}

Ahora viene una cuestión fundamental: el array de poemas lo tenemos en el ViewModel. Esto no es conveniente ya que cada Activity tiene su propia copia del ViewModel. Es decir la MainActivity y la PoemActivity tendrán dos copias diferentes del PoemsViewModel, y por lo tanto dos copias diferentes del ArrayList.

Además, el ArrayList del PoemsViewModel de la MainActivity estará rellenado con los poemas que se han rellenado en la MainActivity, pero el ArrayList del PoemsViewModel de la PoemActivity estará todavía vacío.

Podría parecer que la solución pasa por rellenar el ArrayList en el PoemsViewModel, y así las dos Activities tendrían los mismos datos. Pero no es así, en realidad tendrían dos ArrayList diferentes con una copia idéntica de los datos. Pero ¿qué pasaría si en la MainActivity modificamos el rating de una Poema? ... en el ArrayList de la PoemActivity no se modificaría dicho rating.

La solución entonces pasa por que los ViewModel obtengan el mismo array de poemas.

Para ello crearemos un clase llamada PoemsRepository (que contendrá el array de poemas), usando el patrón Singleton (static):

public class PoemRepository {

   static PoemRepository poemRepository;
   List<Poem> poemList = new ArrayList<>();

   static PoemRepository getInstance(){
       if(poemRepository == null){
           poemRepository = new PoemRepository();
       }

       return poemRepository;
   }
}

Solo queda modificar el PoemsViewModel, para que obtenga el ArrayList<Poem> del PoemsRepository:

public class PoemsViewModel extends ViewModel {
   List<Poem>  getPoemList(){
       return PoemRepository.getInstance().poemList;
   }
}
